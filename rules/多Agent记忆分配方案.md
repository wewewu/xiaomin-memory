# 多 Agent 记忆分配方案

**创建日期**: 2026-02-19
**场景**: 多 Agent 协作时的记忆空间分配

---

## 分配原则

| 维度 | 目录 | 用途 |
|------|------|------|
| 身份 | agents/ | Agent 专属身份和知识 |
| 交接 | handoffs/ | 任务交接信息 |
| 项目 | projects/ | 项目隔离记忆 |

---

## 目录说明

### 1. agents/ — Agent 身份记忆

**适用场景：**
- Agent 启动时加载
- 记录 Agent 的"人设"
- 长期知识积累

**文件示例：**
```
agents/小智.md      → 身份：实验室研究员，技能：ETH分析
agents/trader.md   → 交易策略、风控规则
```

---

### 2. handoffs/ — 任务交接

**适用场景：**
- 任务完成，需要交接给下一个 Agent
- Agent A 临时无法继续，Agent B 接手
- 上下文需要跨 Agent 传递

**文件示例：**
```
handoffs/小岷→小智.md   → 小岷交给小智的任务
handoffs/小智→小绿.md   → 小智交给小绿的任务
handoffs/紧急切换.md    → 紧急情况交接
```

---

### 3. projects/ — 项目隔离

**适用场景：**
- 多 Agent 协作同一项目
- 需要隔离不同项目的记忆
- 项目专属的上下文

**文件示例：**
```
projects/ETH交易/
├── 小智-分析.md   → 小智的分析记录
├── 小绿-风控.md   → 小绿的风控规则
└── 共享记忆.md    → 共享信息

projects/小智培养/
└── 成长记录.md
```

---

## 实际工作流示例

```
场景：ETH 交易项目

1. 项目启动
   → 创建 projects/ETH交易/

2. Agent 加入
   → agents/小智.md 加载身份
   → agents/trader.md 加载技能

3. 任务交接（小岷 → 小智）
   → handoffs/分析任务.md

4. 分析完成，风控接力
   → handoffs/小智→风控.md

5. 项目结束
   → projects/ETH交易/ 归档
```

---

## 总结

| 问题 | 答案 |
|------|------|
| 什么时候用 agents/? | Agent 的"身份"和"人设" |
| 什么时候用 handoffs/? | 任务在 Agent 之间传递时 |
| 什么时候用 projects/? | 多 Agent 协作的"项目"记忆 |
